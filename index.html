Tutoriels – Déploiement d’un serveur Web sous Linux

1. Installer Apache sur Ubuntu

Apache HTTP Server est l’un des serveurs web les plus utilisés. Il permet d’héberger des sites web accessibles via un navigateur. 

sudo apt update
sudo apt install apache2 -y
sudo systemctl start apache2
sudo systemctl enable apache2

Après l’installation, Apache écoute par défaut sur le port 80. L’accès se fait via l’adresse IP de la machine Ubuntu.

2. Tester le serveur Web

Pour vérifier le bon fonctionnement du serveur, ouvrez un navigateur et entrez l’adresse IP de la machine Ubuntu : 

http://ADRESSE_IP_SERVEUR

L’apparition de la page par défaut Apache confirme que le serveur est opérationnel.

3. Héberger un site web sous Ubuntu

Le dossier par défaut des sites web Apache est : 

/var/www/html

Pour remplacer la page par défaut par votre propre site : 

sudo rm /var/www/html/index.html
sudo nano /var/www/html/index.html

Insérez votre code HTML puis sauvegardez.

4. Transfert du site web depuis Windows vers Ubuntu (VM)

Le site web est développé sur la machine hôte Windows. Il doit être transféré vers la machine virtuelle Ubuntu.

Méthode 1 : Dossier partagé (recommandée)

Dans VirtualBox, configurez un dossier partagé entre Windows et Ubuntu. 

/media/sf_nom_du_dossier
        
sudo cp -r /media/sf_siteweb/* /var/www/html/
sudo chown -R www-data:www-data /var/www/html

Méthode 2 : SCP (ligne de commande) 

scp -r siteweb utilisateur@IP_UBUNTU:/var/www/html

5. Conclusion du tutoriel

À travers ces étapes, nous avons mis en place un serveur web fonctionnel sous Linux Ubuntu, hébergé un site web et assuré le transfert des fichiers depuis la machine hôte Windows vers la machine virtuelle.

Documentation Technique

Guide professionnel de déploiement sur Ubuntu Server

5.1 Introduction

Cette partie décrit le déploiement complet d'un site web sur une machine Ubuntu virtuelle, depuis la préparation de l'environnement jusqu'au test final dans un environnement local, sans nécessairement utiliser un nom de domaine.

5.2 Préparation de l'installation d'Ubuntu sur VMware

  • Télécharger l'ISO d'Ubuntu (version LTS recommandée)
  • Créer une nouvelle VM VMware (2 CPU, 4 Go RAM, 20 Go disque)
  • Configurer le réseau en mode "Bridged" pour un accès direct
  • Installer Ubuntu puis mettre à jour le système après le premier démarrage

5.3 Configuration du serveur (Apache2)

5.3.1 Mise à jour du système
sudo apt update
sudo apt upgrade -y
5.3.2 Installation d'Apache2
sudo apt install apache2 -y
sudo systemctl enable apache2
sudo systemctl start apache2
5.3.3 Configuration du pare-feu
sudo ufw allow 'Apache Full'
sudo ufw enable
sudo ufw status

5.4 Configuration du site web

5.4.1 Autorisation du contenu
sudo chown -R www-data:www-data /var/www/html
sudo chmod -R 755 /var/www/html
5.4.2 Déplacer le dossier du site web
sudo rm -rf /var/www/html/*
sudo cp -r /chemin/vers/site/* /var/www/html/
5.4.3 Configuration dans le terminal
sudo nano /etc/apache2/sites-available/000-default.conf
sudo apache2ctl configtest
sudo systemctl reload apache2

5.5 Test dans le navigateur avec l'adresse IP ou un nom local

  • Ouvrir http://IP_DU_SERVEUR pour valider l'accès local
  • Si besoin, utiliser une adresse IP ou un nom simple pour l'accès depuis le réseau
  • Tester http://adresse-ip-ou-nom-local après configuration

5.6 Conclusion

Le serveur web est opérationnel, le site est déployé et accessible via l'adresse IP, un accès local ou, si nécessaire, un nom de domaine simple.

1. Architecture Client – Serveur

L’architecture client-serveur est le modèle fondamental du Web. Le client (navigateur web) envoie une requête HTTP vers un serveur web qui traite cette requête et renvoie une réponse.

Client (Navigateur)
    │ Requête HTTP
    ▼
Serveur Web (Apache / Nginx)
    │ Réponse HTTP
    ▼
Client (Affichage de la page)

Ce modèle garantit la séparation entre l’interface utilisateur et le traitement serveur.

2. Le protocole HTTP

HTTP (HyperText Transfer Protocol) est le protocole utilisé pour la communication entre le client et le serveur web.

  • GET : demander une ressource (page web)
  • POST : envoyer des données au serveur
  • PUT : mettre à jour une ressource
  • DELETE : supprimer une ressource

Les réponses HTTP contiennent un code de statut (200, 404, 500…) indiquant le résultat de la requête.

3. Installation d’Apache sous Linux

Apache HTTP Server est l’un des serveurs web les plus utilisés sous Linux. Son installation se fait via le gestionnaire de paquets. 

# Mise à jour du système
sudo apt update

# Installation d'Apache
sudo apt install apache2

# Démarrage du service Apache
sudo systemctl start apache2

# Activation au démarrage
sudo systemctl enable apache2

Après installation, Apache est accessible par défaut via l’adresse http://localhost ou l’adresse IP du serveur.

4. Structure des fichiers Apache

Apache utilise une organisation de fichiers standard sous Linux : 

/etc/apache2/
├── apache2.conf
├── ports.conf
├── sites-available/
├── sites-enabled/
/var/www/html/
  • /etc/apache2 : fichiers de configuration
  • sites-available : configurations des sites
  • sites-enabled : sites actifs
  • /var/www/html : dossier par défaut du site web

5. Sécurité du serveur web

La sécurisation d’un serveur web est essentielle pour protéger les données et garantir la disponibilité du service.

  • Activation du pare-feu (UFW)
  • Utilisation du protocole HTTPS (SSL/TLS)
  • Mise à jour régulière du système
  • Restriction des permissions fichiers
# Autoriser Apache dans le pare-feu
sudo ufw allow 'Apache Full'

# Vérifier l'état du pare-feu
sudo ufw status

Comparatif des Serveurs Web

Apache HTTP Server

Performance : Moyenne

Consommation : Élevée

Avantages : Très flexible, grande communauté, nombreux modules.

Inconvénients : Moins performant sous forte charge.

Usage : Sites dynamiques, serveurs éducatifs et professionnels.

Nginx

Performance : Très élevée

Consommation : Faible

Avantages : Excellente gestion du trafic, reverse proxy efficace.

Inconvénients : Configuration avancée moins intuitive.

Usage : Sites à fort trafic, applications modernes.

NetWare de Novell

Performance : Moyenne

Consommation : Moyenne

Avantages : Intégré dans les réseaux Novell, bonne sécurité.

Inconvénients : Moins utilisé aujourd'hui, dépendance à l'écosystème Novell.

Usage : Réseaux d'entreprise Novell, serveurs de fichiers.

IIS (Internet Information Services)

Performance : Bonne

Consommation : Moyenne

Avantages : Intégration native avec Windows et .NET.

Inconvénients : Dépendance à l’environnement Windows.

Usage : Applications ASP.NET, infrastructures Microsoft.

Google Web Server (GWS)

Performance : Excellente

Consommation : Optimisée

Avantages : Utilisé par Google, haute disponibilité et scalabilité.

Inconvénients : Non open-source, peu d'informations publiques.

Usage : Infrastructures Google, services web à grande échelle.
